# Работа с изображениями

Markdown позволяет легко вставлять изображения в вашу документацию. Вы можете использовать как внешние изображения, так и те, что хранятся в вашем репозитории.

---

### Внешние изображения

Это изображения, которые находятся на других веб-сайтах. Для их вставки нужна прямая ссылка на само изображение (обычно заканчивается на `.jpg`, `.png`, `.gif` и т.д.).

**Шаблон:**
```markdown
![Альтернативный текст изображения](https://example.com/image.jpg "Подсказка при наведении")
```
*   `Альтернативный текст` — текст, который отображается, если изображение не загрузилось, или для скринридеров.
*   `URL изображения` — прямая ссылка на изображение.
*   `Подсказка при наведении` (необязательно) — текст, который появляется при наведении курсора на изображение.

**Пример:**
**Шаблон:**
```markdown
![Логотип GitHub](https://github.githubassets.com/images/modules/logos_page/GitHub-Mark.png "Логотип GitHub")
```

**Результат:**
![Логотип GitHub](https://github.githubassets.com/images/modules/logos_page/GitHub-Mark.png "Логотип GitHub")

---

### Изображения из репозитория

Для изображений, которые вы загрузили прямо в ваш репозиторий GitHub, используются относительные пути. Рекомендуется хранить изображения в папке `docs/assets/images/`.

**Шаблон:**
```markdown
![Альтернативный текст](/путь/к/изображению/в/репозитории/image.png "Подсказка")
```
*   Путь начинается с `/`, указывая на корень сайта, а затем идет путь к файлу относительно папки `docs/`.

**Пример (предположим, у нас есть `docs/assets/images/local_image.png`):**
**Шаблон:**
```markdown
![Локальное изображение](/assets/images/local_image.png "Изображение из репозитория")
```

**Результат:**
<!-- Здесь должно быть изображение, но я не могу его сгенерировать. Представьте, что здесь загрузилось бы изображение из вашего репозитория. -->
![Локальное изображение](/assets/images/local_image.png "Изображение из репозитория")

### Изображения из Google Docs и Яндекс Документов

**Прямая вставка изображений из Google Docs или Яндекс Документов в Markdown напрямую и надежно НЕ РАБОТАЕТ.**

Эти сервисы не предоставляют прямых статических ссылок на изображения, которые можно было бы просто вставить в Markdown. Вместо этого они генерируют временные или защищенные ссылки, которые:
*   Требуют авторизации пользователя.
*   Могут меняться со временем.
*   Предназначены для отображения внутри самого документа, а не для встраивания на сторонний сайт.

**Обходные пути (не рекомендуются для долгосрочного использования):**
1.  **Скачать изображение:** Самый надежный способ — скачать изображение из Google Docs/Яндекс Документов на свой компьютер, а затем загрузить его в репозиторий (как "Изображение из репозитория" выше).
2.  **Сделать скриншот:** Сделать скриншот и загрузить его в репозиторий.

**Рекомендация:**
Для стабильной и надежной работы всегда скачивайте изображения с этих сервисов и храните их локально в вашем репозитории, а затем ссылайтесь на них как на "Изображения из репозитория".
